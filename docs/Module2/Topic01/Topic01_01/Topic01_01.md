# Topic 1.1 - 面向对象编程简介

## 1. 面向对象编程简介

我们在 Python 基础这门课中所学的编程方法，称为**面向过程编程（Procedural Programming）**

- 这种编程方法的核心思想是将程序视为一系列按顺序执行的步骤，通过函数来组织代码，从而实现特定的功能
- 面向过程编程强调的是程序的流程和步骤，适用于简单的任务和小型程序
- 而当程序变得复杂时，面向过程编程可能会导致代码难以维护和扩展

**面向对象编程（Object-Oriented Programming, OOP）** 是一种更为强大和灵活的编程范式，基本是现代软件的主流编程方法

- 面向对象编程的核心要素就是**类（Class）**和**对象（Object）**
- 其中，对象又可以有**属性（Attribute）**和**方法（Method）**，之前有同学比较疑惑，Python 中可以自定义函数，那怎样能自定义方法呢？
- 因为方法的调用方式是 `对象.方法()`，因此定义方法之前要先定义对象，而定义对象又必须先定义类
- 学了面向对象编程之后，大家就会明白如何自定义方法了

## 2. 面向对象与面向过程的区别

面向过程，更多的是倾向于流程和步骤，强调的是**该做什么了**，当然不一定非得是线性的顺序执行，可以有分支和循环：

- 例如，我们做过的动物园程序，整个代码套在一个无限循环里，然后根据用户输入的选项执行分支语句
- 比方说，用户输入1，就执行应该展示骆驼了，展示完骆驼之后，就该回到主循环的开头了，等等

面向对象，更多的是倾向于对象和行为，强调的是**谁来做什么**：

- 例如，植物大战僵尸这款游戏中，游戏界面中的各个角色，都是对象，它们各自有自己的属性和行为
- 比方说，向日葵就是来生产阳光的，豌豆射手就是来发射豌豆的，僵尸就是来吃植物的

当然，面向过程和面向对象并不是完全对立的，只是侧重点不同而已：

- 我们在动物园程序里，也有对象的概念，比方说动物的文件读取进来之后，每个文件对象调用读取方法后，我们才能看到动物的ASCII图
- 我们在植物大战僵尸这款游戏里，也有一个从头到尾流程的概念，比方说游戏开始时要加载地图，阳光是每隔一段时间产生一次，豌豆要先射出来才能达到僵尸身上，等等

## 3. 面向对象编程的三大特性

面向对象编程有三个重要的特性，分别是**封装（Encapsulation）**、**继承（Inheritance）**和**多态（Polymorphism）**，这三个概念我们会在后续的课程中详细介绍，这里先做个简介：

(1) **封装**的概念其实大家已经有所了解了

- 大家之前写过函数，函数就是一种封装，把一段代码封装成一个整体，通过调用函数名就可以执行这段代码
- 在面向对象编程中，我们定义类，并把这个类的属性和方法封装在一起，通过创建对象来使用这些属性和方法，也是一种封装的思想
- 比方说，植物大战僵尸这款游戏中，向日葵这个类封装了它的属性（例如生命值）和方法（例如生产阳光）

(2) **继承**是指一个类可以继承另一个类的属性和方法，从而实现代码的重用和扩展

- 比方说，植物大战僵尸这款游戏中有不同种类的僵尸，例如普通僵尸、铁桶僵尸、跳跃僵尸
- 但是其实这些僵尸都是大同小异的，比方说它们都有生命值、移动速度等属性，也都有走路、攻击等方法
- 所以，我们就可以先定义一个“僵尸”类，当中定义好僵尸的通用属性和方法，如果我想再定义一个铁桶僵尸，你会发现只要在“僵尸”类的基础上，加一个铁桶属性就行了，而不需要重新定义所有的属性和方法

(3) **多态**是指不同的类可以有相同的方法名，但实现的功能可以不同

- 比方说，植物大战僵尸这款游戏中，豌豆射手、双发射手、食人花，这三种植物都有一个“攻击”方法
- 只不过它们的攻击方式不同，豌豆射手每次发射一颗豌豆，双发射手每次发射两颗豌豆，食人花是用嘴巴咬人
- 但是，它们调用的都是同一个方法名“攻击”，这就是多态的体现
